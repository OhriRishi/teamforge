# FTC TeamForge v1.4.0 - Surveys Feature

**Release Date:** 2025-01-22

## Overview

This release introduces a comprehensive Surveys feature, enabling teams to create, publish, and analyze surveys for feedback collection. Surveys support multiple question types, customizable templates with branding, and flexible visibility controls for different audience types.

## New Features

### Survey Creation & Management
- **Create surveys**: Build surveys with title, description, and multiple questions
- **Question types**: Support for 6 question types:
  - Short answer (single line text)
  - Long answer (multi-line text)
  - Multiple choice (single selection)
  - Dropdown (single selection from dropdown)
  - Checkboxes (multiple selection)
  - Rating (star rating 1-5)
- **Question ordering**: Drag questions to reorder them
- **Required questions**: Mark questions as required or optional
- **Survey status**: Draft, Published, or Closed workflows

### Survey Templates
- **Reusable templates**: Create branded templates for consistent survey appearance
- **Header images**: Upload custom header images for surveys (5MB max, JPEG/PNG/GIF/WebP)
- **Footer content**: Add rich text footer content using BlockNote editor
- **Template management**: Create, edit, and delete templates per team

### Survey Visibility Controls
- **Public surveys**: Anyone can view and respond (no login required)
- **Private surveys**: Requires authentication to view and respond
- **Team surveys**: Only team members can view and respond

### Response Collection & Analytics
- **Public response collection**: Share survey links for anonymous responses
- **Response tracking**: View all responses with respondent info (name, email if provided)
- **Response analytics**: Aggregate response data and statistics
- **Individual response view**: View detailed answers per response
- **Confirmation pages**: Respondents see confirmation after submission

## Database Changes

### New Tables

| Table | Description |
|-------|-------------|
| `survey_templates` | Reusable survey templates with branding |
| `surveys` | Survey definitions with status and visibility |
| `survey_questions` | Questions with type, options, and ordering |
| `survey_responses` | Response submissions with respondent info |
| `survey_answers` | Individual answers to questions |

### New Columns
- `surveys.visibility` - Controls who can access the survey (public/private/team)
- `surveys.template_id` - Optional reference to a survey template

### New Indexes
- `idx_surveys_team_season` - Team and season lookup
- `idx_surveys_status` - Status filtering
- `idx_surveys_visibility` - Visibility filtering
- `idx_surveys_template` - Template lookup
- `idx_survey_questions_survey` - Questions by survey
- `idx_survey_responses_survey` - Responses by survey
- `idx_survey_templates_team` - Templates by team

### New Storage Bucket
- `survey-templates` - Stores header images for survey templates (public, 5MB limit)

### New Functions
- `get_surveys_with_question_count(team_id, season_id)` - Returns surveys with question counts

## Row Level Security

### Survey Templates
- Team members can CRUD their team's templates
- Anyone can view templates used in public published surveys

### Surveys
- Team members can CRUD their team's surveys
- Public published surveys visible to anyone
- Private published surveys require authentication
- Team surveys require team membership

### Survey Questions
- Team members can CRUD questions for their surveys
- Visibility-based read access matching parent survey

### Survey Responses
- Team members can view responses to their surveys
- Anyone can submit responses based on survey visibility
- Respondents can view their own response (confirmation page)

### Survey Answers
- Team members can view answers to their surveys
- Anyone can create answers based on survey visibility

## API Endpoints

### Surveys
- `GET /api/surveys` - List surveys for team/season
- `POST /api/surveys` - Create new survey
- `GET /api/surveys/[surveyId]` - Get survey details
- `PUT /api/surveys/[surveyId]` - Update survey
- `DELETE /api/surveys/[surveyId]` - Delete survey

### Survey Questions
- `POST /api/surveys/[surveyId]/questions` - Add question
- `PUT /api/surveys/[surveyId]/questions/[questionId]` - Update question
- `DELETE /api/surveys/[surveyId]/questions/[questionId]` - Delete question

### Survey Responses
- `GET /api/surveys/[surveyId]/responses` - List responses
- `POST /api/surveys/[surveyId]/responses` - Submit response
- `GET /api/surveys/[surveyId]/responses/[responseId]` - Get response details

### Public Survey Access
- `GET /api/public/surveys/[surveyId]` - Get public survey
- `POST /api/public/surveys/[surveyId]/responses` - Submit public response

### Survey Templates
- `GET /api/survey-templates` - List templates
- `POST /api/survey-templates` - Create template
- `PUT /api/survey-templates/[templateId]` - Update template
- `DELETE /api/survey-templates/[templateId]` - Delete template
- `POST /api/survey-templates/upload` - Upload header image
